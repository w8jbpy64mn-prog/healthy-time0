# Healthy Time | وقت الصحة

تم تحويل المشروع إلى **Web App فعلي قابل للتشغيل محليًا للمعاينة قبل النشر**.

## ما الذي يعمل الآن؟

- رابط إدارة مستقل: `/admin`
- دخول إدارة أساسي (افتراضي):
  - username: `admin`
  - password: `HealthyTime@2026`
- إدارة أولية للعملاء (إنشاء عميل + اشتراك + رابط خاص)
- إنشاء رابط عميل تلقائي لكل اشتراك
- تفعيل/إيقاف رابط العميل من لوحة الإدارة
- صفحة عميل خاصة: `/c/:token`
- إظهار بيانات الاشتراك والعدادات (إجمالي/مستخدم/متبقي)
- اختيار الوجبات مع التحقق من:
  - الحد اليومي للوجبات
  - الحد اليومي للسناكات
  - صلاحية نوع الوجبة حسب نوع الباقة

## التشغيل المحلي (معاينة قبل النشر)

```bash
npm install
npm run dev
```

ثم افتح:
- الإدارة: `http://localhost:3000/admin`
- رابط العميل: يتم توليده من داخل لوحة الإدارة بعد إضافة العميل.

## الملفات الأساسية

- `server.js` — API + منطق الاشتراكات + توليد روابط العملاء.
- `public/admin.html` — واجهة الإدارة.
- `public/customer.html` — واجهة العميل.
- `public/styles.css` — تنسيق الواجهة.
- `data/db.json` — تخزين محلي للبيانات (يُنشأ تلقائيًا).

## ملاحظات مهمة

هذا الإصدار هو **MVP تشغيلي** للمعاينة السريعة قبل النشر، والخطوة التالية تكون:
1. نقل التخزين من JSON إلى PostgreSQL.
2. إضافة نظام صلاحيات متقدم (RBAC).
3. إضافة صفحات التحضير والتغليف والتوصيل بشكل كامل.
4. إضافة اختبارات تكامل وواجهات أكثر احترافية.
